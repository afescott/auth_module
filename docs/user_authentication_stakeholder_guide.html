User Authentication Table

Stakeholder Technical Guide
What This Does: This table stores who can log into your Shopify dashboard and what they're allowed to do. Think of it as your employee/user directory with access control.

Core Identity Fields

Field What It Does Business Value
id Unique identifier for each user Like an employee ID - ensures we never mix up two users
merchant_id Links user to a specific Shopify store Multi-tenancy: Keeps Store A's users separate from Store B's users. Critical for data privacy.
email User's login email How users log in. Also used for password resets, notifications.
display_name Name shown in the app "John Smith" instead of "john.smith@company.com" - better UX

Authentication Fields

Field What It Does Why It Matters
password_hash Encrypted password (NOT plain text) Security: We never store actual passwords. If database is breached, passwords are safe.
shopify_user_id Shopify's ID for this user (if they logged in via Shopify) Flexibility: Users can log in through Shopify OR with email/password. Some users might only use Shopify login.

Access Control

Field What It Does Business Impact
role Permission level: viewer, manager, or admin Security & Compliance:
  • Viewer = Can only look, no changes
  • Manager = Can edit products/orders
  • Admin = Full control, can add/remove users
Prevents junior staff from accidentally deleting critical data.
is_active Whether user can log in Offboarding: When employee leaves, set to false instead of deleting. Preserves audit trail ("who made this change?")

Operational Fields

Field What It Does Business Value
last_login_at When user last logged in
  • Security monitoring: Flag accounts that haven't logged in for 90+ days
  • License optimization: "Are we paying for users who don't use the system?"
  • Audit compliance: "Who had access when?"
created_at When user account was created Audit trail for compliance
updated_at When account was last modified Track when permissions changed

Data Integrity Rules

Rule What It Means Why It's Important
UNIQUE(merchant_id, email) Same email can't be used twice within one store Prevents confusion. But john@company.com can exist for Store A AND Store B (different businesses).
ON DELETE CASCADE If merchant deleted, all their users are deleted Data cleanup: When a store uninstalls your app, automatically remove all their users. No orphaned data.
